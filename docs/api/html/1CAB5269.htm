Remarks

A thread dispatcher provides a prioritized queue for delegates which is processed on a single specified thread.

The delegates are executed in the order of their priority and then in the order they were added to the queue. When all delegates are executed, or the queue is empty respectively, the thread dispatcher waits for new delegates to process.

A watchdog can be used to ensure that the execution of a delegate does not block the dispatcher indefinitely. The watchdog is active whenever WatchdogTimeout is not null. The watchdog runs in a separate thread and raises the Watchdog event if the execution of a delegate takes longer than the specified timeout.

Whether ExecutionContext, SynchronizationContext, and/or CultureInfo is captured from the dispatching thread onto the delegate execution thread (and used for executing a delegate), depends on the used ThreadDispatcherOptions.
