Adds an object to the list of objects which are kept alive at least as long as this thread dispatcher is running.

Remarks

The keep-alive list simply stores a reference to the specified object, preventing the garbage collector from collecting the object. Therefore, the keep-alive list is useful for keeping objects which are needed later, e.g. during the processing of a delegate, but are not required anywhere else. An example for this is a timer which needs to live as long as the timer is running, e.g. a one-shot timer where the timer instance itself (apart from its timer event) is not required anymore after it was created.

 Notes to Implementers
After the thread dispatcher is shut down, the keep-alive list must be cleared.
 Notes to Implementers
Must be callable from the dispatcher thread.
