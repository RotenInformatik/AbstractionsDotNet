Remarks

A SimpleDispatcher provides a queue for delegates which is processed on the thread where Run is called (Run blocks while executing the queue until Shutdown(ThreadDispatcherShutdownMode), ShutdownAsync(ThreadDispatcherShutdownMode), or BeginShutdown(ThreadDispatcherShutdownMode) is called).

The delegates are executed in the order they are added to the queue. When all delegates are executed, or the queue is empty respectively, SimpleDispatcher waits for new delegates to process.

During Run, the current SynchronizationContext is replaced by an instance of ThreadDispatcherSynchronizationContext and restored afterwards.

A watchdog can be used to ensure that the execution of a delegate does not block the dispatcher undetected. The watchdog is active whenever WatchdogTimeout is not null. The watchdog runs in a separate thread and raises the Watchdog event if the execution of a delegate takes longer than the specified timeout.

 Important
Whether ExecutionContext, SynchronizationContext, and/or CultureInfo is captured and used for executing a delegate, depends on the used ThreadDispatcherOptions.
Thread Safety
Static members of this type are safe for multi-threaded operations. Instance members of this type are safe for multi-threaded operations.
